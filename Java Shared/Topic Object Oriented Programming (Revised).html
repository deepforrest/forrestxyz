The Pillars of OOP:
Overview
  1. Abstraction ~ Hiding of data or Information
  2. Encapsulation ~ Binding of data and functions together
  3. Inheritance ~ Allows objects to take on properties/methods of other pre-existing objects
  4. Polymorphism ~ Many forms
Abstraction:
What is Abstraction
  • Reducing the code complexity and redundancy.
  • Providing only needed code for use while hiding wot it works.

Example:
  • Coffee machines
    • You need to know how to use the machine ~ add water add coffee beans.
    • You do not need to know how the coffee machine itself works. 
  • Translated to Java: 
    • You can have a class that represents your coffee machine with a bean/flavor list field and a brew coffee method.
    • To use your coffee machine you would just have to create an object and call its methods.
      • you do not need to know how it selects the coffee or the specific details when brewing the coffee (how much water per cup, how much beans etc..)
Encapsulation:
What is Encapsulation
  • Using access modifiers to restrict modification of data throughout program.
  • Getters and Setters to interact with encapsulated resources.
  • Binding data together in a class supplemental to the OOP paradigm because all of the info about n object is in one place.

Non-Access Modifiers
  • Static
    • Resource available without creating an instance of the class.
    • All static fields are shared among all instances of the class.
      • For example: a counter will have the same value for all instances of the class.
  • Final
    • Makes fields constant ~ Cannot be changed or rewritten.
    • Cant override.
    • Uninheritable.
    • Cannot change the reference point.
  • Abstract
    • Abstract classes cannot be instantiated, only extended (inherited)
    • Abstract methods do not need a body implemented ~ classes that extend the method must implement the body.
  • Transient 
    • Prevents the resource from being serialized.
  • Synchronized
    • Limits the amount of threads that can access a resource at a given time to only one.
  • Default
    • Interface specific
    • Allows you to provide a body for abstract methods.
  • StrictFP
    • If on a class: 
      • All methods are strictFP ~ ensures all floating points are configured to IEEE standards.
  • Volatile
    • Variable must be accessed via the main memory.
    • All threads will be able to access this specific resource regardless of it being local or not.
  • Native
    • Implementation of a method that is marked as native will not be written in Java.
Inheritance:
What is Inheritance
  • Class's ability to pass on or adopt methods and fields from its parent (Parent -> Child Class)
  • Eliminates code redundancy
Polymorphism:
What is Polymorphism
  • Giving multiple forms and uses to an object
    • Overriding - Inherit a method from a parent but its implementation is changed in the child class
      • Method with the same signiture but different implementation
    • Overloading - Inherit a method from a parent but its signature and/or implementation are changed.
