Servlet:
What is a Servlet
  • A servlet is a java application that helps in intercepting requests from the client in order to respond properly.
  • Can be used to connect your front end to your back end.
  • Middleware
  • They are objects that receive a request and send a response back
Servlet & URL
  • The URL pattern requires a particular servlet
    • "http://website.com/index.html"
      • "Website.com" = Domain name (IP address)
      • "index.html" = file
      • http://website.com/HelloWorldServlet
    • allows us to take form data or make an ajax request.
    • NOTE: Servlets come from "javax.servlet.httplibray"
Servlet Interface
  1. HttpServletRequest
  2. HttpServletResponse
  3. HttpServletSession
*Imported from the  "javax.servlet.httplibray" library

Servlet Life-Cycle:
  1. Client sends a request to the server
  2. If the server is on it creates a request and a flat (empty) response file. It then forwards both the request and the response files to the Web Container
  3. The Web Container checks the Deployment Descriptor where to send the request and response
  4. The Web Container wraps the request and response as Java objects
    1. If a servlet is not instantiated, the Web Container invokes the servlets "init()" method.
      1. Servlets are not instantiated until they are needed
    2. The Web Container calls the "service()" method using the request and response objects as inputs. 
      1. Web Controller creates a thread pool to split users into different set
      2. Each pair (request & response) gets its own thread in its thread pool
    3. If the server shuts down or is idle for enough time, the Web Controller calls the "destroy()" method
      1. The "destroy()" method preps the servlet for the Garbage Collector
  5. The servlet passes the request and the newly propagated response object back to the Web Controller
  6. The Web Controller unwraps the request and response and sends them to the server
  7. The server parses the request and response back to the client
Web Container
  • Does a lot of the heavy lifting
  • References an applications Deployment Descriptor, which is a configuration file that tells the server how we want to deploy the content
  • Our Deployment Descriptor will be the "web.xml", which contains info about the servlet mapping
**request = HttpServletRequest /// Response = HttpServletResponse
Sending Users to Different Locations on the Web:
  • Servlets take requests from clients and forward them to requested resources.
  • How? 
    • Send Redirect
    • Forward
Send Redirect
  • Supplied by the response object
  • It is the name of a method: response.sendRedirect(location)
  • Actually sends a response back to the client and sends a new request back.
  • As a result, the information from the previous request is lost
  • Returns a 300 series status code
Forward
  • Supplied by the request object
  • This method in Java is declared in the RequestDispatcher interface
  • When you are using forward, the request never leaves the server
  • You can't request dispatch into another location
  • You make one request rather than 2
Takeaways
  • Use send redirect if you want to give your client a resource that is in some other location 
    • The request appears in the browser as a new request
  • Use forward if you are passing another resource from the same project
    • Something you have access to immediately
  • Forward is handled internally 
  • Send redirect is handled by the browser
