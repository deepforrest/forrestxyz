Literals
  • Primitives are not objects but instead special data types built into the language. 
  • Source code representation of a fixed value
    • Literals are the values stored in the respective data types.
Strings
  • Objects! Java string class is used to make strings
    • Strings are arrays of characters 
  • Immutable - once instantiated their values cannot be changed
  • Strings implement, Serializable, Comparable and CharSequence interface 
  • Strings live in the String pool. 
  • Strings objects are created by:
    • Assigning a string literal (strings encased in double quotes "") 
      • If the string already exists in the string pool the variable will point to that instead of creating a new string
    • using the new keyword
      • In this case, two string objects will be created. 
      • A normal string object will be placed in the non pool heap area
      • the string literal will be placed in the string pool and the variable will reference it.

Variables:
What Are Variables
  • A name given to a memory location.
  • In Java variables are strongly typed, meaning you need to declare the type of variable as well.
Variable Scopes
  • Defines the section in the program where the variable is visible.
  • Class Scope (Member/instance Variables)
  • Method Scope (Local Variables)
  • Block Scope (Loop Variables)
Casting:
What is Casting
  • Datatype conversion
  • Widening (implicit Casting) / Narrowing (Explicit Casting)
Widening
  • Implicit Casting:
    • This is the conversion of a smaller literal to a larger data type.
    • Java handles this for you.
Widening Example:
Int integer = x;
long Lng;
Lng = integer;
  • Since integer literal is smaller, java automatically converts to the larger datatype.
  • No need to explicitly declare the cast.
    • There is already enough space to accommodate the integer value.
Narrowing
  • Explicit Casting
    • You must declare the new cast type when assigning the new value.
    • (datatype)Variable 
Narrowing Example:
long Lng = x
int integer;
int = (int)Lng
  • The data size of long is much bigger. You must explicitly tell java that you are casting to int in this case
Wrappers:
What is a wrapper
  • Java provides Wrapper Classes, which are object representations of the eight primitive types
    • A class that can encase a primitive data type in their respective object class
  • Data Structures that are in the Collection framework only accept objects and thus a primitive must be wrapped in order to pass it.
Autoboxing vs Auto-Unboxing
  • Autoboxing:
    • When passing a primitive to a method/execution that requires a Wrapper Class as input it will implicitly wrap the primitive into its Wrapper Class without you having to explicitly do so.
  • Auto-Unboxing
    • Like Autoboxing, when passing a wrapper to a method/execution that excepts a primitive, it will implicitly convert to the primitive type.
