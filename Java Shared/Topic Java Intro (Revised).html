Notes in Depth
Java:
What is Java
  • An open source programming language
  • A computing platform used by many different software and technologies 
    • Web, Phones (Android Development), Desktops, and Enterprise platforms.
    • It has a wide range of use and is a well known language.
  • It  is a general purpose language and uses Object Oriented Programming paradigm.
  • Launched by Sun Microsystems team in 1995 and based of the basic library.
Pros:
  • Object Oriented.
  • It is platform independent. (Write once run anywhere)
  • High level language with a simple syntax and mild learning curve.
    • High level meaning that it requires a compiler to convert it into machine readable code. 
  • It is a known standard for enterprise level software.
  • It is equipped with security precautions and measures. 
  • Automatic memory management. 
  • Multi-Threading

Cons
  • Oracle announced that they will start charging for Java Standard Edition 8 starting as of 2019 for production.
  • Java is notorious for poor performance. 
    • No built in memory management (garbage collector can take up to 20% of CPU usage)
  • Lacking amount of GUI features to choose from
  • Verbose/Noisy when code is complex
JVM/JRE/JDK
JVM
  • Java Virtual Machine
    • Converts byte code to machine readable code
    • Independent from hardware and software
      • This is what makes Java so portable
  • Heart of the Java Language and provides platform independence
JRE
  • Java Runtime Environment
    • Contains supporting libraries 
    • Implementation of the JVM
  • Running Java Code
JDK
  • Java Development Kit
    • Contains all the developer tools, executable and binaries required to compile
      • Javac
    • Converts Java code to binary for distribution.
    • Implementation of the JVM and JRE
  • Development
Compilation:
  1. Java is converted into byte code with the use of the Javac compiler (JDK)
  2. Files are saved on disk as a '.class' file
  3. When it is time to run the JIT (Just in Time) Compiler is used to convert from byte code to machine code. 
    • Works with the JVM
  4. The JVM converts to native language and the code is executed.
Java Memory Structure:
Heap
  • Stores objects
  • Contains the string pool
  • Has two parts or 'generations'
    • Nursery - allocation for new objects
      • Further broken down into three parts:
        • Eden
        • 2 survivor memory
    • Old Space/Generation
Heap Lifecycle
  1. New objects are added to the nursery and stored in the Eden space.
  2. When Eden is filled, the minor garbage collector (gc) is called and survivor objects are moved to one of the survivor spaces
  3. The minor GC also checks the survivor space and moves objects to the other survivor space so that one space is always empty.
  4. Objects that have survived multiple cycles of GC are added to the old generation
  5. When old generation is full, the major GC is called and memory is reallocated 
Stack
  • Stores local variables and function calls
  • Contains references to the objects in the heap
