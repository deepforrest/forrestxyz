Questions/Answers
  1. What is Java?
    • A programming language.
    • Object Oriented
  2. What is the difference between JRE, JDK, and JVM?
    • JVM = Java Virtual Machine. When we compile code, it is converted into machine language. It is the structure that will execute byte code
    • JRE = Java Runtime Environment. This is what is required to run Java code. It contains the JVM and supporting libraries
    • JDK = Java Development Kit. The software development environment used to develop java applications. It contains the JRE and thus the JVM as well as developer tools (such as the compiler and archiver)
  3. What happens during the compilation process?
    • When we compile Java code the the JAVAC compiler converts it into byte code and saves it as a ".class" file. When the code is executed the JIT (Just-in-Time) compiler converts it into machine readable code. 
  4. What are the benefits of Java?
    • Java is an Object Oriented language. This allows for easy to read and easy to reuse coding practices.
    • Its main advantage is its portability on all operating systems. (The JVM acts as a translator)
  5.  What is a constructor
    • A constructor is a block of code (not a method) that resembles a method however does not have a return type. A constructor must have the same name as its class. It is used to initialize a new object.
  6. What are the primitive datatypes?
    • Int, byte, long, short, float, double, char, boolean
  7. What is a no args constructor?
    • A constructor that takes no arguments. It is usually provided implicitly by Java if no constructor is created prior to compilation.
  8. What is the default constructor?
    • A constructor that is implicitly provided if no constructors are created for a class. It is a simple no argument constructor.
  9. What are the scopes of a variable in java?
    • Instance Scope = Any field within a class but outside of a method. It is created with the object and is destroyed with the object.
    • Method Scope = Any field created within a method. Its life cycle ends when the method stops executing
    • Loop Scope = Like a method scope, fields created within the loop exist in that loop and are destroyed once the loop is complete.
      • Nested loops are within the parent scope
    • Static Scope = The most accessible scope. Fields are created when the app starts and destroyed when the app stops.

    • Default. Access = Class, Sub-Class (within package) and Package
    • Protected. Access = Class, Sub-Class (Within any package) and Package
    • Public. Access = No restrictions
    • Private. Access = Class
  11. Non access modifiers
    • Static, Final, Abstract, Default, Transient, Synchronized, Strictfp, Volatile, Native

  22. What is the string pool?
    • Memory space located in the heap memory. 
    • This is where string literals are stored.
  23. What is the difference between the stack and the heap?
    • The Stack = Memory space for Local and Reference variables and method indications. It executes in a structured manner (Last on = First off) so that when executing code the last block of code added to the stack resolves first. (Main method will be at the bottom)
    • The Heap = Memory space for instance variables and objects. The heap also contains the string pool where string literals live.

  39. What are the pillars of object oriented programming? explain them.
    • Encapsulation:
      • Using access modifiers to restrict modification of data throughout the program 
      • enforces the use of getters and setters
      • binds data together into related structures such as classes. All of the info of an object is in one place
    • Abstraction:
      • Reduction in complexity of code. 
      • Eliminates redundancy
      • Allows for implementation of code with out needing a full understanding of how that code works
    • Inheritance
      • A class's ability to pass on or adopt methods/fields from parent to child. 
      • Helps by removing redundancy in coding.
    • Polymorphism:
      • "To have multiple forms"
      • Overriding: Inherit methods from a parent with the ability to change the body of that method. (Has the same signature)
      • Overloading: Inherits methods from a parent with the ability to change the signature of the method. This can be done by changing the parameter type or by adding/removing parameters
      • Coherence: Ability to store data as a parent/child object. All dogs for example are Animals.
