Spring Model View Controller (MVC):
MVC
  • Model View Controller
  • Model => The data being passed, received and manipulated
  • View => what will be displayed
  • Controller => handles logic routing
***
In spring your Front Controller is a servlet already written for you you called the Dispatcher (see image above)
Dispatcher Servlet
  • Spring MVC is request driven and each request is managed by the Dispatcher Servlet
  • Ingratiated with IOC Container
  • Implements Front Controller DP
Annotations
  • @Controller
    • Stereotype annotations
    • Allows restful opperations
  • @RequestMapping
    • Tells Spring the URI that needs to be mapped
    • Also indicates the HTTP method
Handler Mapping:
Handler Mapping
  • Acts like the Request Dispatcher that we need to support our method
  • Tells Dispatcher Servlet which controller to use
Request Mapping Handler Mapping
  • Internal resource view resolver
    • Simplify the String representing the view that the controller return
    • Configuration in XML
Context Code Listener
  • Ties the lifecycle of the app context with the servlet container
  • Automates the creation of the AC so that tomcat will start it for us
  • Configured in Web XML
3 More Scopes
  • Web Scopes
    • Request ~ Scoped to one HTTP request
    • Session ~ Scoped to single session - server side
    • Global Scope ~ Scoped to servlet container
