Notes in Depth
Database:
What are Databases: 
  • Organised collection of data.
    • Stored in an organised format.
  • Allows us to input, manage, organize and retrieve data quickly.
  • Organizes data into tables with:
    • Records = rows
    • Fields = columns
  • Data is the intersection of rows (Records) and columns(Fields).
Relational Database Management Systems
  • Essentially it is a database management system that upholds specified relationships between tables.
  • Includes functions that maintain the security, accuracy integrity and consistency of the data.
SQL:
What is SQL
  • Structured Query Language.
  • The standard language for relational database systems (RDS).
  • There are many dialects:
    • MySQL, PosGress, Oracle SQL, MS SQL, etc.
Sub Languages
  • There are 5 sub languages.
  • Data Definition Language (DDL)
    • Sub-language for creation and altering of table structure.
    • Core Comands
      • CREATE
      • ALTER ~ change existing
      • DROP ~ delete
      • TRUNCATE
  • Data Manipulation Language (DML)
    • pertains to the changing and modification of data.
    • Operates in one specific table.
    • Core commands: (CRUD ~ Create, Read, Update, Delete)
      • INSERT
      • SELECT
      • UPDATE
      • DELETE
  • Data Query Language (DQL)
    • SELECT statements.
  • Data Control Language (DCL)
    • Regulates access to data.
      • Provides security.
      • Imposes restrictions (user permissions).
      • Core commands
        • GRANT - gives permission
        • REVOKE - removes permissions
  • Transaction Control Language (TCL)
    • Pertaining to work on the database and finalize/reverting changes.
    • Core actions:
      • COMMIT - persists data
      • ROLLBACK - reset/revert back to previous save point
      • SAVEPOINT - create a state to roll back too
      • SET_TRANSACTION - set isolation level of the transaction
What are Constraints
  • Constraints and rules to follow for enforcing database table relationships
  • There are 6 Relationships:
    • Primary Key ~ Data in this column is Unique, Not Null and uniquely identifies all records in the table.
    • Foreign Key ~
      • Any column in the table referencing the primary key of another table.
      • Establishes relationships between two columns between the same table OR between different tables
    • Unique ~ No duplicate values in a column (Null is not considered duplicate)
    • Not Null ~ No null values in the column
    • Default ~ Default the column to some specified value
    • Check ~ Adds an extra condition on inserted data
Composite Key
  • A primary key that consists of more than one column is a composite key
What is Cardinality
  • Describes numerical relationships between two tables
  • There are 3 types:
    • one to one, one to many, many to many
One to One (1:1)
  • For every one table you can have one counter part.
    • Can only have on other
  • Tables are only associated with one other table
  • Can be created by using a primary key and a unique foreign key.
One to Many (1:n)
  • A primary key linked to a non-unique foreign key
  • Example: A beehive table to bees
    • There is only one beehive but there can be many bees
Many to Many (n:n)
  • Two tables have such a relationship where many records have a relationship with many records from another table
  • Foreign key not appropriate ~ instead we use a join table [see below for joins]
    • Example: There are many Pokemons that share many moves
Entity Relationship Diagram
  • Commonly refereed to as ERD 
  • A diagram of tables listing their relationships
Joins:
What are Joins
  • A "Join" is what we use to bring the table of two tables together
  • Join tables are useful in describing many to many relationships
  • A safe way to join tables is by using foreign keys from the two tables.
  • There are 6 joins:
    • Inner Join
    • Full Outer Joins
    • Right Join
    • Left Join
    • Cross Joins
    • Self Joins
Inner Join
  • Only shows records with compared values existing in both tables
Full Outer Join
  • Often refered to as "Full Outer Join", "Full Join", "Outer Join"
  • Properly join all data together
  • Substitutes nulls where data is missing.
Right/Left Join
  • Right Join:
    • Returns all records from the right (2nd) table and matched records from the left (1st) table.
    • Nulls are substituted where there is absent data.
  • Left Join:
    • Returns all records from the left (1st table) and matched records from the right (2nd) table.
    • Nulls are substituted where there is absent data.
Cross Join
  • Cartesian cross product of records from both tables
  • Returns a list of permutations of the tables
 
Self Join
  • Join from one table onto itself.
