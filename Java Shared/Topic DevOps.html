DevOps Overview:
DevOps
  • Methodology which combineds development (writing code) ad operations (maintaining infrastructure to manage that code)
  • The goal is to produce more efficient high quality code
Continuous Integration
  • Process of continuously merging code into central repository and checking if it integrates well with code base
  • Make changes to code base
    • Push to git hub/lab automatically
    • Building and running tests on code
  • Tools = Jenkins or Travis
Continuous Delivery
  • Produce software in short cycles.
  • Aims at building, testing, and releasing software with greater speed and frequency.\
  • reduces the cost, time, and risk of implementing changes.
    • Smaller incremental updates rather than one release.
    • Straight forward and repeatable
*Continuous Integration = 1 (Automated)
*Continuous Delivery = 1, 2, 3, 4 (Automated)
*Continuous Deployment = 1 - 5 (Automated)

Delivery Model:
SAAS
Software as a Service
  • Software distribution model where applications are hosted by a vendor or service provider and made available to customers over a network.
  • Providers host software commercially.
  • EX:
    • google docs
    • MS Office
  • Used by: End User
PAAS
Platform as a Service
  • Cloud provider delivers hardware and software tools usually for purpose of app development.
    • Environment for an app is provided.
  • Customers develop the app.
  • EX:
    • Elastic Bean Stalk
    • MS Azure
    • Heroku
  • Used by: Developers
IAAS
Infrastructure as a Service
  • Third party provider hosts hardware, software storage and other components
  • IAAS providers handle system maintenance, backup and resilience planning
  • Resources can be adjusted on command
  • EX:
    • EC2
    • RDS
